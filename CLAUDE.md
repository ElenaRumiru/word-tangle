Этот файл — контракт сессии. Загружается автоматически. Прочти его целиком до начала работы.

## Миссия

Идеально выполнить тестовое задание на позицию гейм-дизайнера/продюсера (Balance & Economy).
Оригинал задания: `Тестовое задание.png`. Краткая выжимка:

1. **Мобильная адаптация журнальной игры Jumble** — UX и управление под мобильные платформы.
2. **10 уровней** с картинками (картинки — нейросетью). Сложность растёт так, будто 10-й уровень — это 100-й уровень в игре.
3. **Интерактивный веб-прототип** (HTML) с **обязательными читами** для тестирования.
4. **Логика генератора уровней** (в общих чертах; расчёт на массовую генерацию в короткие сроки).
5. **Таблица баланса 100 уровней** — параметры сложности и их изменение (сами уровни не нужны).
6. **Scroll Puzzle (Multicast Games)** — перевод с чисто рекламной на гибридную монетизацию:
   гипотезы → внедрение/тестирование → последовательность апдейтов → постановка задачи команде
   (кратко, не ТЗ) → метрики/ивенты и ожидаемые результаты → расчёт на выдуманных данных.

## Правила работы

- **Язык:** документы и UI прототипа — русский; код, имена файлов, ключи данных — английский.
- **Каждый deliverable закрывается только по чек-листу** из своего брифа в `docs/03-briefs/`.
- Прототип обязан работать открытием одного HTML-файла без сборки и сети (картинки — base64 или рядом).
- Читы в прототипе — не опция, а требование задания. Прототип без чит-панели не сдаётся.
- Картинки генерируем через **Higgsfield MCP** (`generate_image`, модель `gpt_image_2`) —
  рецепт и стайл-гайд в [docs/03-briefs/brief-art-generation.md](docs/03-briefs/brief-art-generation.md).
  В промптах НЕ упоминать «Jumble» как бренд — модель дорисовывает трейдмарк-логотип.
- Смоук прототипа — через Playwright MCP (открыть файл, пройти уровень читом и честно, проверить все читы).
- Уровни как данные: JSON по схеме из brief-levels-difficulty.md, встраиваются в HTML.
- Не выдумывать факты о Scroll Puzzle — использовать `docs/01-research/scroll-puzzle-research.md`;
  чего там нет — помечать как допущение.
- Git: репозиторий не инициализирован; если инициализируешь — коммить по фазам, без force-push.
