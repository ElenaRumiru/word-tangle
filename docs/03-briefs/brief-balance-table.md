# Бриф: таблица баланса 100 уровней

Исполнитель: balance-economist.
Deliverables: `deliverables/balance/balance-100-levels.xlsx` (+ csv) и `balance-notes.md`.
Инструмент: скилл xlsx (openpyxl) — таблица с формулами/условным форматированием и графиком кривой.

## Состав таблицы (100 строк, по строке на уровень)

Колонки (система P1–P8 из brief-levels-difficulty — единая с уровнями прототипа):

| Колонка | Тип | Комментарий |
|---|---|---|
| level | 1–100 | |
| words_count (P1) | 1–5 | ступени: 1–2 (lvl 1–3), 3 (4–15), 4 (16–60), 5 (61–100) — уточнить |
| word_len_min / word_len_max (P2) | 4–8 | |
| zipf_band (P3) | 1–5 | 1 = топ-1000 словаря, 5 = редкая лексика |
| scramble_score (P4) | 0–1 | целевое значение для генератора |
| trap_anagrams (P5) | 0–2 | число слов-ловушек с ложной анаграммой |
| answer_letters / answer_words (P6) | 4–14 / 1–3 | |
| pun_type (P7) | direct / idiom / homophone / double | когнитивная сложность кульминации |
| cartoon_transparency (P8) | 1–3 | 3 = почти показывает ответ |
| free_hint_worth | монеты | «щедрость» уровня: сколько валюты игрок получает/тратит в норме |
| target_solve_time_sec | число | медиана ожидаемого времени |
| target_hint_usage | % | доля игроков, тратящих ≥1 подсказку |
| difficulty_score | формула | взвешенная сумма нормированных P1–P8 (веса — в balance-notes) |
| curve_role | ramp / spike / relief / milestone | драматургия |
| notes | текст | что нового появляется на уровне |

## Требования к кривой

1. **Тренд**: difficulty_score растёт сублинейно (например `D(n) = D1 + a·n^0.8`) — быстрый рост
   в начале ощущается сильнее, чем тот же прирост в конце.
2. **Пила**: локальная амплитуда ±10–15% вокруг тренда; после каждого spike — relief.
3. **Майлстоуны**: каждый 10-й уровень — spike + новинка (новый pun_type, первая ловушка,
   первый 3-словный ответ…) — колонка notes обязана показывать «расписание новинок».
4. **Онбординг**: уровни 1–3 — гарантированный успех (см. research: без ловушек, топ-лексика,
   solve time < 60 сек).
5. **Согласование с прототипом**: строки 1, 12, 23, 34, 45, 56, 67, 78, 89, 100 == уровни 1–10
   прототипа (те же значения параметров). Это прямая проверка ревьюером — расхождений быть не должно.
6. **График**: лист с чартом difficulty_score(level) — тренд + пила видны глазами.

## balance-notes.md

- Формула difficulty_score и веса (и почему такие: вклад параметра ≈ его влияние на solve time).
- Логика каждой ступени параметров («words_count растёт на 16-м уровне, потому что…»).
- Как кривая связана с экономикой подсказок и (в полной игре) с местами рекламы/офферов —
  1 абзац, мостик к монетизационному мышлению.
- Ограничения модели и что калибровали бы плейтестом (target_solve_time — гипотеза до данных).
