# Бриф: логика генератора уровней

Исполнители: word-game-designer + balance-economist.
Deliverable: `deliverables/levels/level-generator-design.md`.

Задание: «в общих чертах», но цель — показать инженерную зрелость: пайплайн, который реально
генерирует **тысячи уровней в короткие сроки** с контролируемой сложностью. Псевдокод/схема — да,
рабочий код — не требуется (маленький proof-of-concept скрипт приветствуется, если дёшев).

## Скелет пайплайна (раскрыть каждый шаг)

```
Корпуса данных → Банк ответов-каламбуров → Подбор слов под кружки (CSP) →
Генерация скрэмблов под целевой скрэмбл-скор → Автовалидация →
Промпт карикатуры + батч-генерация картинок → Скоринг сложности →
Раскладка по слотам кривой → Выборочная ручная QA (10%)
```

1. **Корпуса**: частотный словарь с Zipf-рангами (wordfreq/SUBTLEX для EN, НКРЯ для RU),
   стоп-лист (профанити, бренды), банк идиом/омофонов.
2. **Банк ответов** — узкое место всего пайплайна, честно это сказать: каламбуры не генерируются
   комбинаторикой. Решение: LLM-батч (промпт: тема+тип каламбура+длина → N кандидатов) →
   автофильтры (длина, словарность, дубликаты) → ручная модерация топа. Оценить производительность:
   один модератор отсматривает сотни кандидатов в день; LLM-стоимость копеечная.
3. **Подбор слов под кружки** — задача satisfaction: мультисет букв ответа надо покрыть кружками
   K слов заданной длины и Zipf-банда. Жадный алгоритм с backtracking по словарю; кэш «буква →
   слова банда с этой буквой». Показать, что решений много (пространство слов ≫ ограничений).
4. **Скрэмблы**: перестановки с целевым скрэмбл-скором (формула из brief-levels-difficulty);
   ограничения: скрэмбл не словарное слово, не совпадает с ответом, для лёгких уровней сохранять
   биграммы/первую букву, для сложных — максимизировать дезориентацию + опционально псевдослово.
5. **Автовалидация**: единственность анаграммы (или осознанная ловушка), покрытие кружков,
   дубли уровней, профанити в скрэмблах (случайная перестановка может сложиться в нехорошее слово —
   проверять и её!).
6. **Картинки**: шаблонный промпт (стайл-гайд art-брифа) заполняется LLM по ответу+подписи;
   батч в image-API; автопроверки (нет текста-мусора — vision-моделью), стоимость ~$0.005–0.02/шт.
7. **Скоринг сложности**: difficulty_score = взвешенная сумма P1–P8 (веса из balance-notes);
   уровень кладётся в слот кривой с ближайшим целевым скором (пила и передышки — из баланс-таблицы).
8. **Ручная QA**: сэмпл 10%, критерии (решаемость, смешно ли, картинка намекает).

## Обязательно оценить числа

- Пропускная способность: слова+скрэмблы+валидация — тысячи уровней/час на ноутбуке;
  боттлнеки — банк каламбуров (модерация) и картинки (генерация+ревью). Итоговая оценка
  вида «100 уровней/день силами 1 дизайнера-модератора + автоматика» с расчётом.
- Стоимость на 1000 уровней (LLM + image API + часы модерации).
