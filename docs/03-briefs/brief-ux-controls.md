# Бриф: UX и управление мобильной адаптации Jumble

Исполнитель: word-game-designer. Deliverable: `deliverables/levels/ux-concept.md`.
Перед работой прочитать: `01-research/jumble-analysis.md`, `01-research/mobile-word-games-ux.md`.

## Рамка

Целевая платформа: смартфон, portrait, one-handed. Аудитория casual word-games (30+, «уютный» жанр —
без таймеров в кор-лупе). Сессия: 1 уровень = 1–4 минуты.

## Ключевые предрешения (обоснованы исследованием — можно оспорить, но аргументированно)

1. **Сохраняем двухфазную структуру оригинала** (4 слова → ответ по карикатуре), в отличие от Just Jumble,
   который её сломал и получил главную претензию фанатов. Двухфазность — это встроенный ритм
   «разминка → кульминация», и именно она делает игру Jumble, а не generic-анаграммой.
   Мобильная поправка: слова решаются **по одному** (фокус на одном скрэмбле на экране),
   решённые сворачиваются в компактную полосу сверху.
2. **Ввод: tap-to-place** (эталон жанра для «собери конкретное слово из его же букв»):
   тап по плитке пула → буква занимает первый свободный слот; тап по слоту → буква возвращается.
   Обязателен бесплатный **Shuffle**. Drag и QWERTY — отклонены (см. research, табл. ввода).
3. **Карикатура — центр кульминации**: во время фазы слов показана размыто/частично (тизер),
   раскрывается полностью к фазе ответа. Это и интрига, и естественная награда.
4. **Кружки оригинала сохраняются**: в решённом слове буквы-кружки подсвечены и «улетают»
   в буфер ответа — игрок видит, как копится материал для финала.
5. **Без таймера** в базовом режиме (антипаттерн уютного жанра).

## Что должно быть в UX-концепте

1. **Схема экранов**: Home → Level map/select → Game (фаза слов → фаза ответа) → Win → следующий.
   Для каждого — маркап/ASCII-скетч с зонами большого пальца (нижняя треть = интеракции).
2. **Детальный флоу уровня**: старт (тизер карикатуры) → слово 1..N (tap-to-place, кружки улетают) →
   раскрытие карикатуры + подпись-подводка → сборка ответа из накопленных букв (тот же tap-to-place,
   слоты с разбивкой по словам, кавычки каламбура как в журнале) → победная сцена.
3. **Подсказки** (лестница по информативности, из research):
   - Shuffle — бесплатно, всегда;
   - «Открыть букву» — дёшево;
   - «Буква на выбранной позиции» — дороже;
   - «Объяснение шутки» (нарративный хинт а-ля David's Hints) — для фазы ответа;
   - Решить слово целиком — дорого/rewarded ad в полной версии.
   В прототипе подсказки бесплатны, но UI и счётчик валюты присутствуют.
4. **Фидбек и джус** (без реализации сложной анимации в прототипе — но заложить в спеку):
   pop плитки, shake на ошибку, каскад при вер­ном слове, полёт кружков в буфер, конфетти на ответе,
   эскалация похвал (ОТЛИЧНО! → БЛЕСТЯЩЕ!).
5. **Онбординг**: уровень 1 = kinesthetic-туториал без текстовых правил (палец-призрак, слово из 4 букв).
6. **Ошибки и застревание**: неверное слово при заполнении всех слотов — shake + мягкая подсветка;
   fail-состояния нет (уровень нельзя проиграть), затык лечится подсказками.
7. **Доступность**: плитки ≥44pt, контраст, что происходит при длинных словах (перенос пула в 2 ряда).
