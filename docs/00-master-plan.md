# Мастер-план выполнения тестового задания

Статус: фаза 0 завершена (2026-07-07, Fable). Фазы 1–7 выполняет Opus.

## Решения (приняты пользователем 2026-07-07) — РЕШЕНО

1. **Язык контента уровней — АНГЛИЙСКИЙ.** Слова и ответы-каламбуры на английском, опираемся на
   97 готовых пазлов журнала (`01-research/jumble-solutions-raw.txt`) как на сырьё и эталон качества
   каламбуров. **UI прототипа — русский** (кнопки, читы, подсказки, тексты интерфейса). Валидатор
   уровней использует английский частотный словарь (wordfreq) для банд Zipf и проверки анаграмм.
2. **Стиль картинок — ЦВЕТНАЯ МОБИЛЬНАЯ карикатура.** Flat-цвет, дружелюбно, читается на 6".
   Шаблон промпта — «цветной вариант» из `03-briefs/brief-art-generation.md`. По-прежнему без
   логотипов/заголовков (`no title, no logo, no watermark`), aspect_ratio 3:4.

## Принципы качества (напоминание себе)

- Ревьюер оценивает мышление гейм-дизайнера: каждое решение — с «почему», связанным с мобильным UX
  или экономикой. Не «сделал», а «сделал, потому что».
- Сложность — это система, а не «слова длиннее»: см. 8 параметров в brief-levels-difficulty.md.
- Числа в монетизации должны сходиться (ARPDAU = ad + IAP, проверяемая арифметика).
- Краткость постановки задачи команде — сама по себе оцениваемый навык: одна страница, не ТЗ.
