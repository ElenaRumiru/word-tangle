## Детали проверки

### 1. Higgsfield (генерация изображений) — ✔
- `balance`: 1170 кредитов, план ultimate — хватает с большим запасом (10 картинок × ~0.5–1 = ~5–10 кредитов).
- Тест-генерация: ч/б газетная карикатура «человек по щиколотку в море, двое в костюмах торопят» —
  результат в `test-cartoon-gpt-image-2.png`, попадание в стиль оригинального Jumble отличное.
- **Важное наблюдение:** модель самовольно дорисовала трейдмарк-шапку «JUMBLE by David L. Hoyt…».
  → В боевых промптах добавлять `no title, no logo, no header text, no watermark` (учтено в art-брифе).
- Рабочий рецепт (params): `model=gpt_image_2, aspect_ratio=3:4, quality=medium, count=1`;
  забор результата — `job_status(sync:true)` → `rawUrl` → `curl`.

### 2. PDF-пайплайн — ✔
- Установлены `pypdf pdfplumber pypdfium2` (Python 3.11).
- Отрендерены страницы журнала в JPG (scale 1.5), извлечён текст всех страниц решений.
- Результат разбора: `01-research/jumble-analysis.md` + `01-research/jumble-solutions-raw.txt` (97 пазлов).

### 3. Playwright (смоук) — ✔
- `browser_resize(390,844)` → мобильный вьюпорт ставится.
- `browser_navigate(data:text/html…)` → data-URL открывается (значит, откроет и локальный файл).
- `browser_evaluate` → клик по кнопке и чтение DOM (`textContent` → «CHEAT OK») работают.
- Вывод: связка «открыть файл → прожать кнопку/чит → проверить состояние» готова для чек-листа QA.
- Примечание: `browser_click` требует `ref` из snapshot; для читов надёжнее `browser_evaluate`
  с прямым селектором (учтено — QA будет использовать evaluate для проверки чит-функций).

### 4. openpyxl (Excel) — ✔
- Версия 3.1.5; формулы в ячейках и `openpyxl.chart.LineChart` доступны.
- Значит, difficulty_score можно считать формулой в самом файле (требование баланс-брифа), не хардкодом.

## Риски инструментов и запасные варианты

- **Higgsfield лимит/ошибки генерации** → запас кредитов большой; при сбое конкретной картинки —
  регенерация с уточнением промпта (count:2 и отбор).
- **RU-текст в bubble картинки может артефачить** → проверить на уровне 1; если плохо — убрать текст
  из картинки, реплику показывать в UI прототипа (учтено в art-брифе).
- **Размер HTML с 10 base64-картинками** → сжать в jpeg/webp q70 скриптом перед встраиванием;
  цель — файл, который и открывается локально, и проходит как Artifact.
- **Artifact CSP** → прототип не делает внешних запросов (всё инлайн), поэтому совместим; финально
  подтвердить публикацией в фазе 3.
